# DATA: Входные и выходные адреса (A)

## Зависимости
- PLC/01_SYNTAX_BASE.md — базовый синтаксис
- PLC/02_VARIABLES.md — переменные

---

## Общее описание

Адреса A используются для обмена дискретными сигналами между УЧПУ и внешним оборудованием станка (кнопки, переключатели, датчики, реле, контакторы и т.д.). Каждый адрес представляет собой 8-битную ячейку (один байт), где каждый бит соответствует одному дискретному сигналу.

---

## Структура адресов

### Выходные адреса (A1-A16)
| Диапазон | Назначение |
|----------|------------|
| A1-A16 | Дискретные выходы (управление реле, контакторами, индикацией) |

### Входные адреса (A17-A99)
| Диапазон | Назначение |
|----------|------------|
| A17-A99 | Дискретные входы (кнопки, переключатели, датчики) |

*Первый адрес входов (A17) может быть изменён в файле machine.cfg. Последний адрес выходов равен первому адресу входов минус один.*

---

## Формат адресации

### Полный адрес
Aномер.бит

### Нумерация битов
Биты нумеруются от 0 (младший) до 7 (старший).

| Бит | Вес | Маска |
|-----|-----|-------|
| 0 | 1 | 1 |
| 1 | 2 | 2 |
| 2 | 4 | 4 |
| 3 | 8 | 8 |
| 4 | 16 | 16 |
| 5 | 32 | 32 |
| 6 | 64 | 64 |
| 7 | 128 | 128 |

## Использование в ПЭС

### Чтение входного сигнала
if A17.0 = 1
    ; сигнал активен
fi

### Установка выходного сигнала
A1.0 = 1        ; включить
A1.0 = 0        ; выключить

### Работа с байтом целиком
A1 = 255        ; установить все биты A1
A1 = 0          ; сбросить все биты A1

### Битовая адресация
A17.0 = 1       ; установить бит 0
A17.1 = 0       ; сбросить бит 1
A17.2 = 1       ; установить бит 2

### Маскированная запись
A1 & 1 = 1      ; установить только бит 0
A1 & 2 = 0      ; сбросить только бит 1

## Примеры использования

### Пример 1: Обработка аварийного выключателя
:216 RT(5)
    if A19.0 = 0
        error "Аварийный останов"
        d10 = 0
        d49 = 0
        A1.0 = 0
        A1.1 = 0
        A1.2 = 0
    fi

### Пример 2: Обработка кнопок пуска/стопа
:1
    if A18.6 = 1          ; кнопка "Пуск"
        d10 = 1
    fi
    if A18.7 = 1          ; кнопка "Стоп"
        d10 = 0
        d49 = 0
    fi
    wait(100)
    goto 1

### Пример 3: Управление СОЖ
:108                      ; M8 - включение СОЖ
    A3.0 = 1
    kill

:109                      ; M9 - выключение СОЖ
    A3.0 = 0
    kill

### Пример 4: Управление приводом шпинделя
:103                      ; M3 - вращение вперёд
    A2.0 = 1
    A2.1 = 0
    kill

:104                      ; M4 - вращение назад
    A2.0 = 0
    A2.1 = 1
    kill

:105                      ; M5 - останов
    A2.0 = 0
    A2.1 = 0
    kill

### Пример 5: Проверка датчика зоны останова
:205                      ; выход в ноль по X
    while A21.0 = 0
        wait
    wend
    ; датчик сработал
    kill

### Пример 6: Управление зажимом патрона
:110                      ; M10 - зажим
    A3.3 = 1
    A3.4 = 0
    wait(500)
    if A22.0 = 0
        error "Патрон не зажат"
    fi
    kill

:111                      ; M11 - разжим
    A3.3 = 0
    A3.4 = 1
    wait(500)
    if A22.1 = 0
        error "Патрон не разжат"
    fi
    kill

### Пример 7: Индикация состояния
:1
    ; индикация готовности
    if m30 = 255
        A4.2 = 1         ; лампа "Готовность"
    else
        A4.2 = 0
    fi
    
    ; индикация ошибки
    if d39 != 0
        A4.1 = 1         ; лампа "Ошибка"
    else
        A4.1 = 0
    fi
    
    wait(100)
    goto 1

### Пример 8: Маскированная запись (только бит 0)
A1 & 1 = 1               ; установить бит 0, остальные без изменений
A1 & 1 = 0               ; сбросить бит 0

### Пример 9: Присвоение группе
A1, 4 = 0                ; очистить A1-A4
A1.0, 4 = 1              ; установить бит 0 в A1-A4

## Рекомендации

1. **При отладке** можно принудительно устанавливать входные адреса для имитации сигналов

2. **Неиспользуемые выходы** рекомендуется держать в сброшенном состоянии

3. **Для диагностики** входных сигналов используйте страницу "Входные адреса" на экране УЧПУ

4. **В RT-подпрограммах** для максимального быстродействия используйте "быстрые" входы/выходы, сконфигурированные в конфигураторе

5. **При работе с маховиками** используйте соответствующие входы, сконфигурированные как энкодеры
