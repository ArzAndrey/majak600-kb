# HMI: Базовый синтаксис

> **⚠️ ВАЖНО ДЛЯ ИИ:**
> Язык разметки УЧПУ "Маяк-600" использует **фигурные скобки `{ }`** для всех конструкций.
> Закрывающая конструкция повторя открывающую: `{ form ... } form`, `{ view ... } view`, `{ item ... }`.
> **НЕ ИСПОЛЬЗУЙТЕ** синтаксис других ЧПУ (Fanuc, Siemens, обобщённый HMI).
> Все примеры в этом файле — точный синтаксис "Маяк-600".


## Зависимости
- `CONFIG/03_COLORS.md` — цветовые константы
- `CONFIG/04_CONSTANTS.md` — числовые константы (GAP_H, GAP_V)

---

## Система координат
(0,0) — левый верхний угол экрана
X — вправо (пиксели)
Y — вниз (пиксели)


**Размеры экрана:**
- 10.4" диагональ: 800 × 600 пикселей
- 15" диагональ: 1024 × 768 пикселей

## Цвета

Цвета задаются двумя способами:

### 1. Константа из colors.dat
bgr = MAIN_BGR
fgr = AN_C
invbgr = Error_C

### 2. RGB в формате RGB(R,G,B)
color = RGB(255, 205, 0)
bgr = RGB(0, 40, 108)

**Основные цветовые константы:**
| Константа | Назначение |
|-----------|------------|
| MAIN_BGR | Основной фон (тёмно-синий) |
| MAIN_TEXT | Основной текст (светлый) |
| MENU_BGR | Фон меню |
| MENU_FGR | Текст меню |
| AN_C | Цвет символов в режимах "Автомат", "Наладка" |
| REG_F | Фон прямоугольников |
| Error_C | Цвет ошибки (красный) |
| GOT_C | Цвет готовности (зелёный) |
| Messag_C | Цвет сообщений (жёлтый) |
| GREEN | RGB(85, 231, 99) |
| YELLOW | RGB(242, 226, 77) |
| RED | RGB(255, 73, 84) |
| BLUE | RGB(64, 117, 255) |

## Константы

### Определение
#ИМЯ_КОНСТАНТЫ = значение
#ИМЯ_ЦВЕТА = RGB(R,G,B)

### Примеры
#GAP_H = 19
#GAP_V = 16
#FONTSIZE_MID = 20
#MAIN_BGR = RGB(0, 40, 108)

### Использование
Константы можно использовать в любом месте вместо числовых значений:
{ item # () GAP_H GAP_V yes yes 'Автомат' }

**Основные константы из defaults.dat:**
| Константа | Значение | Назначение |
|-----------|----------|------------|
| GAP_H | 19 | Горизонтальный отступ |
| GAP_V | 16 | Вертикальный отступ |
| FONTSIZE_SMALL | 14 | Малый шрифт |
| FONTSIZE_MID | 20 | Средний шрифт |
| FONTSIZE_BIG | 19 | Большой шрифт |
| HEIGHT_MID | 17 | Высота строки для среднего шрифта |
| BOX_X | 506 | Координата X блоков (F, S, и т.д.) |
| BOX_W | 142 | Ширина блоков |

## Объявление переменных

Для использования переменных в диалоговых окнах и действиях их необходимо предварительно объявить.

**Синтаксис:**
{ variable имя_переменной1 имя_переменной2 ... }

**Описание:**
Блок `{ variable }` объявляет глобальные переменные диалога, которые могут использоваться в формах HMI (в элементах `item`), в действиях `action`. Объявление должно быть расположено до их первого использования.

**Пример:**
{ variable PRNvar }
{ variable Nvar }
{ variable Gvar }
{ variable Xvar }
{ variable Zvar }
{ variable Fvar }

**Использование в элементе item:**
{ item Xvar (height=24 ratio=0.7 width=70) 320 90 no yes 'X[+9.3]' }

**Использование в action:**
{ action xvar_set
   { X %=(x<-9999999)||(x>9999999) ifok 6 %(Xvar=x) 0 "Неверное значение X" errormsg}
}

**Примечания:**
- Переменные могут хранить числовые значения
- Специальное значение NAN (-1) означает "не определено"
- В элементах item формат вывода задается в квадратных скобках, например '[+9.3]'

## Комментарии

Однострочные комментарии — символ ; до конца строки.

; Это комментарий
{ item # () GAP_H GAP_V yes yes 'Автомат' } ; Комментарий после кода

## Вложенные файлы

Для подключения внешних файлов используется конструкция @имя_файла@:

@clock.dat@
@box_name_s.dat@
@m28form.dat@

**Важно:** Имена подключаемых файлов и их расширения могут различаться в зависимости от конкретной реализации HMI. Файлы могут иметь расширения .dat, .hmi, .mnu, .txt или другие. При работе с конкретным проектом необходимо ориентироваться на фактические имена файлов, указанные в коде.

Вложенные файлы могут содержать любые конструкции языка (item, view и т.д.).

## Блок default (параметры по умолчанию)

Блок { default ... } default задаёт параметры для всего файла. Параметры, указанные в default, можно не указывать в элементах.

{ default
    xmax = 800
    ymax = 600
    height = FONTSIZE_MID
    fgr = MAIN_TEXT
    bgr = MAIN_BGR
    align = TEXT_LEFT
} default

**Параметры, которые можно задать в default:**
| Параметр | Описание |
|----------|----------|
| xmax | Ширина доступной области (пиксели) |
| ymax | Высота доступной области (пиксели) |
| height | Высота шрифта (пиксели) |
| fgr | Цвет текста |
| bgr | Цвет фона |
| invfgr | Цвет текста в инверсии |
| invbgr | Цвет фона в инверсии |
| align | Выравнивание (TEXT_LEFT, TEXT_RIGHT, TEXT_CENTERED) |
| border | Цвет рамки |
| fonts | Название гарнитуры |

---

## Выравнивание текста

| Константа | Значение | Назначение |
|-----------|----------|------------|
| TEXT_LEFT | 1 | По левому краю |
| TEXT_RIGHT | 2 | По правому краю |
| TEXT_CENTERED | 3 | По центру |

Можно комбинировать через +:
align = right + bottom   ; правый нижний угол

## Блок form (форма)

Определяет форму (экран). Номер формы используется для переключения между экранами.

Синтаксис:
{ form <номер_формы> (атрибуты)
    { view ... } view
    { menu ... } menu
} form

Атрибуты:
- menu=<номер_меню> — связывает форму с меню
- xmax=<пиксели> — максимальная ширина формы
- ymax=<пиксели> — максимальная высота формы

Пример:
{ form 0 (menu=10, xmax=667, ymax=600)
    { hotkey KEY_ENTER { pressed d_set_ust } }
    { view ()
        { item # () GAP_H GAP_V yes yes 'Автомат' }
    } view
} form

## Блок view (слой отображения)

Определяет "слой" или режим отображения на форме. Позволяет переключать отображение различной информации на одном экране. Обычно внутри одной формы используется несколько блоков view, между которыми можно переключаться (например, клавишами F9/F10 или стрелками).

Синтаксис:
{ view (атрибуты)
    элементы (item, box, picture и т.д.)
} view

Атрибуты:
- fgr=<цвет> — цвет текста по умолчанию
- bgr=<цвет> — цвет фона по умолчанию
- invfgr=<цвет>, invbgr=<цвет> — цвета для инвертированного текста (например, при вводе или выделении)
- xmax=<пиксели>, ymax=<пиксели> — размеры области просмотра
- windowed=1 — если указан, view ведет себя как окно со скроллингом
- menu=<номер_меню> — связывает данный view со своим меню (меню может переключаться вместе со view)
- page=0 — если указан, данный view не будет участвовать в циклическом перелистывании (Page Up / Page Down)

Пример:
{ view (fgr=AN_C bgr=REG_F invfgr=REG_F invbgr=AN_C xmax=800 ymax=600)
    @clock.dat@
    { item # () GAP_H GAP_V yes yes 'Автомат' }
    { item # (invbgr=Error_C invfgr=MAIN_BGR width=105) 456 16 yes no ''   
        { if m30=0 invfgr=MAIN_BGR format=' ЧПУ: Авария  ' } }
    { item # (invbgr=Error_C invfgr=MAIN_BGR width=106) 456 16 yes no ''
        { if m30=255 invbgr=GREEN format=' ЧПУ: Готово  ' } }
} view

## Блок grid (таблицы и диалоги)

Блок { grid ... } grid используется для создания таблиц, диалоговых окон и многостраничных интерфейсов. Каждый grid представляет отдельную страницу и может содержать свои view, menu, hotkey и элементы.

Синтаксис:
{ grid <имя> (menu=<номер_меню>)
    { hotkey KEY_ENTER { pressed <действие> } }
    { view ... } view
    { menu ... } menu
} grid

Атрибуты:
- menu=<номер_меню> — связывает страницу сетки с меню

Пример:
{ grid c01 (menu=10001)
    { hotkey KEY_ENTER { pressed xvar_set_sel } }
    { view (fgr=AN_C bgr=REG_F invfgr=REG_F invbgr=AN_C xmax=800 ymax=600)
        { picture abc="picf.cfg" pic="q" scale=1 solid=1 bgr=-1 windowed=1 x=10 y=60 width=-1 height=-1 }
        { item # (height=HZG ratio=RZG fgr=NAME_C) XZG YZG no yes 'Линейная интерполяция G1 ' }
        { item Nvar (width=70 height=HPR ratio=RPR) XPR YPR+HPR*2 no yes 'N[5]' 
            !"<xvar>@N"#"<xvar>" { if Nvar=NAN format='N ---' } }
    } view
} grid

## Блок item (элемент вывода/ввода)

Основной элемент для вывода информации и ввода данных. Подробно описан в файле HMI/03_ELEMENTS.md.

Синтаксис:
{ item <источник_данных> (атрибуты)
    <x> <y> <индикация> <нуль-индикация> '<формат>'
    { if <условие> ... }
    +<кнопка_увеличения> -<кнопка_уменьшения>
    !"<команда>"#"<подсказка>"
}

Источник данных может быть:
- # — статический текст
- m67, i34, d77, p11 — переменные
- m32.0 — бит M-переменной
- time — системное время
- diskfree — свободное место на диске

Атрибуты:
- height=<значение> — высота (пиксели)
- width=<значение> — ширина (пиксели)
- ratio=<число> — коэффициент сжатия шрифта
- fgr=<цвет> — цвет текста
- bgr=<цвет> — цвет фона
- invfgr=<цвет> — цвет текста при инверсии
- invbgr=<цвет> — цвет фона при инверсии
- align=<выравнивание> — TEXT_LEFT, TEXT_RIGHT, TEXT_CENTERED, RIGHT+BOTTOM
- fonts='<имя_шрифта>' — например, 'Mayak-Icons'

---

## Блок box (прямоугольная рамка)

Элемент для рисования прямоугольной рамки.

Атрибуты:
- color=<цвет> — цвет линии рамки
- x=<X>, y=<Y> — координаты верхнего левого угла
- x2=<ширина>, y2=<высота> — ширина и высота

Пример:
{ box color=B6 x=BOX_X y=BOX_F_Y x2=BOX_W y2=BOX_F_H }

## Блок picture (изображения)

Используется для вставки графических изображений в диалоговых окнах.

Атрибуты:
- abc="<файл_конфигурации>" — файл с настройками рисунков
- pic="<идентификатор>" — идентификатор рисунка в конфигурации
- scale=1 — масштабирование (1 — без масштабирования)
- solid=1 — заливка
- bgr=-1 — прозрачный фон (-1 = прозрачный, цвет = фоновый)
- windowed=1 — оконный режим
- x=<X>, y=<Y>, width=<ширина>, height=<высота> — позиция и размер

Пример:
{ picture abc="picf.cfg" pic="q" scale=1 solid=1 bgr=-1 windowed=1 x=10 y=60 width=-1 height=-1 }

## Блок variable (переменные диалога)

Объявляет переменные, которые используются в диалоговых окнах для временного хранения данных перед записью в кадр УП. Переменные могут хранить значения параметров циклов (X, Y, Z, F, S и т.д.). Эти переменные существуют **только в контексте HMI** и недоступны для ПЭС.

Синтаксис:
{ variable <имя_переменной> }

Пример:
{ variable PRNvar }
{ variable Nvar }
{ variable Gvar }
{ variable Xvar }
{ variable Yvar }
{ variable Zvar }
{ variable Fvar }
{ variable Svar }

### Присвоение значения переменной

Значение присваивается с помощью конструкции %(Имя=значение):

%(Nvar=100)          ; присвоить Nvar значение 100
%(Xvar=50.5)         ; присвоить Xvar значение 50.5
%(PRNvar=1)          ; установить режим записи

Присвоение может использоваться внутри действий (action), обработчиков нажатий (pressed) или инициализации.

### Использование переменной в выводе

Переменная выводится на экран с помощью формата в атрибуте item:

{ item Nvar (height=HPR width=WID_P) X Y no yes 'N[5]' }

Здесь [5] означает вывод переменной как целого числа шириной 5 знаков.

{ item Xvar (width=80) X Y no yes 'X[+9.3]' }

Формат [+9.3] выводит число со знаком, 9 знаков до запятой, 3 после.

### Использование переменной в условиях

Переменные могут использоваться в условных конструкциях для динамического изменения отображения:

{ item # ... 'Статус'
   { if PRNvar=0 format=' Режим ввода ' }
   { if PRNvar=1 format=' Режим замены ' }
}

{ item # ... 'Текст'
   { if Nvar=NAN format='N ---' }
}

### Кнопки увеличения/уменьшения

Для переменных можно создавать кнопки изменения значения:

+ff10 -ff20   ; увеличить/уменьшить значение на 10/20
+I10 -I20     ; увеличение/уменьшение Ivar на 10/20

При нажатии на кнопку с + значение переменной увеличивается, с - — уменьшается.

### Значение "не задано" (NAN)

Для проверки, задано ли значение переменной, используется константа NAN. Обычно NAN определяется в начале файла:

#NAN=-1
{ default nan=NAN }

### Пример использования переменных

{ variable PRNvar }
{ variable Nvar }
{ variable Xvar }
{ variable Zvar }
{ variable Fvar }

{ action g1new
   c " Nvar=N,0" cc blockscatter
   %(Nvar=Nvar+1)
   %(Gvar=1)
}

{ grid c01 (menu=10001)
   { view ...
      { item Nvar (width=70) X Y no yes 'N[5]' }
      { item Xvar (width=70) X Y no yes 'X[+9.3]'
         { if Xvar=NAN format='X ---' } }
      { item Zvar (width=70) X Y no yes 'Z[+9.3]'
         { if Zvar=NAN format='Z ---' } }
   } view
} grid

## Блок action (действия)

Определяет действие, которое может быть вызвано при нажатии на кнопку меню, выборе элемента или по горячей клавише. Действие содержит последовательность команд, которые могут изменять переменные, вызывать другие действия, формировать кадры УП.

Синтаксис:
{ action <имя_действия>
    { <параметр> <команда> ... }
    { <другой_параметр> <команда> ... }
} action

Пример:
{ action xvar_set_sel c "<?xvar>" cc parse ifok 3 "Ошибка!" errormsg 0 xvar_set }

{ action gs1 %GSvar=91 0 }   ; по стрелкам G90/G91
{ action gs2 %GSvar=90 0 }

{ action c01new inpop  
    c " Nvar=N,0" cc blockscatter
    %(Nvar=Nvar+1)
    %(Gvar=1)
    %(GSvar=90) }

## Блок hotkey (горячие клавиши)

Привязывает действие или команду к комбинации клавиш.

Синтаксис:
{ hotkey KEY_<комбинация> { pressed <действие_или_команда> } }

Доступные комбинации:
- KEY_ENTER — Enter
- KEY_CTRL_E — Ctrl+E
- KEY_ALT_S — Alt+S
- KEY_ALT_0 — Alt+0
- KEY_ALT_1 — Alt+1
- KEY_ALT_2 — Alt+2
- KEY_ALT_6 — Alt+6
- KEY_ALT_7 — Alt+7
- KEY_ALT_8 — Alt+8
- KEY_CTRL_PGUP — Ctrl+PageUp
- KEY_INS — Insert

Примеры:
{ hotkey KEY_CTRL_E { pressed safetxted } }
{ hotkey KEY_ALT_S { pressed %=(m33>1) ifok 2 g_access 0 g_mfuncs } }
{ hotkey KEY_ALT_0 { pressed !s0-0 form m } }   ; Переход в режим "Автомат"
{ hotkey KEY_ALT_1 { pressed !s1-0 form m } }   ; Переход в режим "Ручной"
{ hotkey KEY_ALT_2 { pressed !s2-0 form m } }   ; Переход в режим "Выход в ноль"

## Блок menu (меню)

Определяет меню, которое отображается в нижней части экрана. Содержит до 14 кнопок f1-f14.

Синтаксис:
{ menu <номер_меню>
    { f1 { text "Текст кнопки" } { pressed <действие> } }
    { f2 ... }
    ...
} menu

Пример:
{ menu 10001
    { f1 { text "Записать$в УП" { if PRNvar=1 text="Записать$кадр" } } 
        { pressed inpop c " N=Nvar G=Gvar G=GSvar X=Xvar. Y=Yvar. Z=Zvar. F=Fvar"
                  cc blockgather 
                  %=(PRNvar#1) ifok 3 quit xload 0
                  inpush quit "<?ed>" insorrepl } }
    { f6 { text "Возврат" } { pressed indropall quit } }
    { f15 { text "Возврат" } { pressed indropall quit } }
} menu
