# HMI: Меню

> **⚠️ ВАЖНО ДЛЯ ИИ:**
> На экране УЧПУ "Маяк-600" всегда присутствуют два ряда меню: горизонтальный (F1-F6) и вертикальный (F7-F14).
> Меню описывается как `{ menu N { f1 ... f14 } } menu`, а не `menu 0 ... horizontal`.
> **НЕ ИСПОЛЬЗУЙТЕ** синтаксис других ЧПУ — в "Маяк-600" все кнопки нумеруются f1-f14.
> Все примеры в этом файле — точный синтаксис "Маяк-600".

## Общее описание

На экране всегда присутствуют два ряда меню:
- **Горизонтальный ряд** — клавиши F1–F6 (отображаются в нижней части экрана)
- **Вертикальный ряд** — клавиши F7–F14 (отображаются справа)

Специальные клавиши для расширения меню:
- **Ctrl** — альтернативный набор функций (ctrlF1–ctrlF6, ctrlF7–ctrlF14)
- **Alt** — альтернативный набор функций (altF1–altF6, altF7–altF14)

Смена меню происходит независимо от остальных зон экрана. Меню привязывается к форме или view через атрибут `menu=<номер_меню>`.

---

## Синтаксис меню

{ menu <номер_меню> [атрибуты]
    { f1 [атрибуты]
        { text "Текст" [атрибуты] { if <условие> ... } }
        { pressed <действие> }
        { released <действие> }
    }
    { f2 ... }
    { f3 ... }
    ...
    { f14 ... }
} menu

### Горизонтальные клавиши (F1–F6)
- f1, f2, f3, f4, f5, f6 — описание клавиш горизонтального меню
- ctrlf1 ... ctrlf6 — при нажатой клавише Ctrl
- altf1 ... altf6 — при нажатой клавише Alt

### Вертикальные клавиши (F7–F14)
- f7, f8, f9, f10, f11, f12, f13, f14 — описание клавиш вертикального меню
- ctrlf7 ... ctrlf14 — при нажатой клавише Ctrl
- altf7 ... altf14 — при нажатой клавише Alt

Если клавиша не используется, её можно пропустить.

## Атрибуты меню

Атрибуты указываются в фигурных скобках после номера меню.

| Атрибут | Описание |
|---------|----------|
| `hiddenfkeymenu=1` | Скрыть строку подсказок для горизонтальных клавиш F1-F6 на экране. Позволяет использовать эти клавиши для скрытых функций, не занимая место на экране. |

### Пример
{ menu 0 (hiddenfkeymenu=1)
    { f1 { text "Ручной" } { pressed !s1-0 form mm } }
    { f2 { text "Автомат" } { pressed !s0-0 form mm } }
} menu

## Структура кнопки

### Атрибуты кнопки
Атрибуты указываются в фигурных скобках после номера кнопки.

| Атрибут | Описание |
|---------|----------|
| `access=<уровень>` | Определяет, при каком уровне доступа клавиша будет доступна. Например, `access=-1` — доступна только при отрицательном уровне (специальные права) |
| `hiddenfkeymenu=<0/1>` | Если установлен в 1, строка подсказок для этой клавиши не отображается (может также задаваться в блоке default) |

### Блок text
Определяет текст, отображаемый на кнопке.

**Синтаксис:**
{ text "Текст" [атрибуты] { if <условие> ... } }

**Специальные символы в тексте:**
| Символ | Значение |
|--------|----------|
| `$` | Разрыв строки (перенос текста на следующую строку кнопки) |
| `#номер` | Выбор варианта текста по номеру (используется с условиями) |

**Атрибуты text:**
| Атрибут | Описание |
|---------|----------|
| `fgr=<цвет>` | Цвет текста |
| `bgr=<цвет>` | Цвет фона |

#### Условное изменение текста кнопки (if внутри text)

Блок `{ if <условие> text="Текст" }` внутри `{ text ... }` позволяет динамически изменять надпись на кнопке в зависимости от значения переменной. Это особенно полезно для кнопок, которые меняют свое назначение в зависимости от режима или состояния.

**Синтаксис:**
{ text "Текст по умолчанию" { if <условие> text="Альтернативный текст" } }

**Примеры:**

; Переключение надписи в зависимости от режима записи
{ f1
    { text "Записать$в УП" { if PRNvar=1 text="Записать$кадр" } }
    { pressed ... }
}

; Изменение цвета текста в зависимости от состояния
{ f2
    { text "СОЖ$вкл" (fgr=RGB(150,150,150)) { if (d154#1) fgr=RGB(255,255,0) } }
    { pressed %=d154#1 ifok 2 d154=0 0 d154=1 }
}

; Сложное условие с несколькими вариантами
{ f4
    { text "Пред.$G-функция" { if PAGEvar#NAN fgr=REG_F } }
    { pressed ... }
}

**Множественные условия в одном text:**
Можно использовать несколько блоков `{ if ... }` внутри одного `{ text ... }` для изменения разных атрибутов:

{ f5
    { text "След.$G-функция" 
        { if PAGEvar#NAN fgr=REG_F }
        { if PAGEvar=NAN fgr=MAIN_BGR }
    }
    { pressed ... }
}

#### Условное отображение текста (#номер_варианта)

Позволяет выводить разные строки в зависимости от значения переменной. Это альтернатива использованию нескольких блоков `text` с условиями.

Синтаксис: `#номер_варианта` или `#` для базового варианта.

Пример:
{ f1
    { text "#1" (fgr=RGB(150,150,150)) { if d311=0 fgr=RGB(255,255,0) } }
    { text "вкл" (fgr=RGB(150,150,150)) { if d311=1 fgr=RGB(255,255,0) } }
    { pressed %=d311#1 ifok 3 d311=1 0 d311=0 }
}
Здесь `#1` означает, что выводится текст, соответствующий значению переменной d311.

### Блок pressed
Определяет действие, выполняемое при нажатии на кнопку.

**Синтаксис:**
{ pressed <действие> }

### Блок released (необязательный)
Определяет действие, выполняемое при отжатии кнопки.

**Синтаксис:**
{ released <действие> }

## Команды pressed / released

### Переключение меню
| Команда | Описание |
|---------|----------|
| `mm` | Переключение на предыдущее меню |
| `mmN` | Переключение на указанное меню (N — номер) |
| `mm10` | Переключение на меню 10 |

### Вызов сетки или модуля
| Команда | Описание |
|---------|----------|
| `pressed имя_сетки` | Вызов сетки |
| `pressed имя_модуля` | Вызов модуля |

### Системные команды
| Команда | Описание |
|---------|----------|
| `!s1-0` | Переключение режима (например, в ручной) |
| `!s0-0` | Переключение режима (например, в автомат) |
| `form` | Возврат к форме |
| `quit` | Закрыть текущий модуль |
| `close` | Закрыть с сохранением |
| `view` | Переключить вид |
| `next` | Переход к следующему |
| `execute` | Выполнить действие |
| `indropall quit` | Очистить наборную строку и выйти |
| `xload` | Загрузить что-либо |
| `inpop` | Вставить в наборную строку |
| `inpush` | Отправить в наборную строку |
| `blockgather` | Собрать блок данных |
| `blockscatter` | Разобрать блок данных |
| `insorrepl` | Вставить или заменить кадр |
| `errormsg` | Вывести сообщение об ошибке |

### Команды с условием (ifok)
Синтаксис:
%=<условие> ifok <число_команд> <команда_1> 0 <команда_2>

- `<условие>` — выражение сравнения
- `<число_команд>` — количество команд (команда_1 + команда_2)
- `<команда_1>` — выполняется, если условие ложно
- `<команда_2>` — выполняется, если условие истинно

Пример:
%=d404#0 ifok 3 "1" start_log 0

### Команды с условием (ifnotok)
Синтаксис:
%=<условие> ifnotok <число_команд> <команда_1> 0 <команда_2>

- `<команда_1>` — выполняется, если условие истинно
- `<команда_2>` — выполняется, если условие ложно

### Другие команды
| Команда | Описание |
|---------|----------|
| `inputkill` | Очистить наборную строку |
| `d130=3` | Задание D-ячейки |
| `d130=3` | Задание параметра |

## Меню переключения режимов

Специальное меню (номер 0), вызываемое клавишей РЕЖИМ (Tab). Использует только горизонтальный ряд.

{ menu 0 ; tab-menu
    { f1 { text "Ручной" } { pressed !s1-0 form mm } }
    { f2 { text "Автомат" } { pressed !s0-0 form mm } }
    { f3 { text "Выход$в ноль" } { pressed !s2-0 form mm } }
    { f4 { text "Выход$в точку" } { pressed !s6-0 form mm } }
    { f5 { text "Фиксир.$перемещ."} { pressed !s8-0 form mm d406=1 } }
    { f6 { text "Преднабор" } { pressed !s7-0 form mm } }
    { altf3 (access=-1) { text "Выполнить" } { pressed execute } }
} menu

## Связывание меню с формой

Меню связывается с формой, view или grid через атрибут `menu=<номер_меню>`.

### Связывание с формой
{ form 0 (menu=10, xmax=667, ymax=600)
    { view () ... } view
} form

### Связывание с view
{ view (menu=20 fgr=AN_C bgr=REG_F) ... } view

### Связывание с grid
{ grid c01 (menu=10001) ... } grid

## Блок default для меню

В блоке default можно задать параметры для всего меню.

{ default
    hiddenfkeymenu = 1   ; скрыть подсказки для всех горизонтальных клавиш
    fgr = MAIN_TEXT      ; цвет текста по умолчанию
    bgr = MENU_BGR       ; цвет фона по умолчанию
} default

## Правила использования меню

1. **Номер меню должен быть уникальным** для каждой формы или view.

2. **Кнопки f1-f6 — горизонтальный ряд**; f7-f14 — вертикальный ряд.

3. **Текст кнопки может быть многострочным** — используйте `$` для переноса.

4. **Условия в меню могут изменять текст и цвет** — это позволяет создавать динамические меню. Используйте синтаксис `{ if условие text="Текст" }` внутри блока `{ text ... }`.

5. **Действие может быть сложным** — можно комбинировать несколько команд.

6. **Используйте `indropall quit` для выхода** — это очищает наборную строку и закрывает форму.

7. **Для переключения между режимами используйте условия с `#номер`** — это стандартный подход для выбора варианта.

8. **Параметр hiddenfkeymenu скрывает строку подсказок** — полезно для скрытых функций.

9. **Ctrl и Alt расширяют функциональность** — используйте ctrlF1, altF1 и т.д. для дополнительных действий.

## Таблица параметров меню

| Параметр | Описание | Где указывается |
|----------|----------|-----------------|
| `access` | Уровень доступа | в описании клавиши: `f1 (access=-1)` |
| `hiddenfkeymenu` | Скрыть подсказки горизонтальных клавиш | в меню: `{ menu 0 (hiddenfkeymenu=1) }` или в default |
| `fgr` | Цвет текста клавиши | в блоке text: `{ text "Текст" (fgr=RED) }` |
| `bgr` | Цвет фона клавиши | в блоке text: `{ text "Текст" (bgr=BLUE) }` |

## Типичные ошибки

| Неправильно | Правильно | Пояснение |
|-------------|-----------|-----------|
| `menu 0 horizontal` | `{ menu 10 ... } menu` | Используйте фигурные скобки и номер меню |
| `{ menu 10 { f1 "Пуск" } }` | `{ menu 10 { f1 { text "Пуск" } { pressed ... } } } menu` | Каждая кнопка должна иметь text и pressed |
| `{ text "Записать в УП" }` | `{ text "Записать$в УП" }` | Для переноса строки используйте `$` |
| `{ pressed action }` без определения | `{ pressed start }` где start определён в action | Действие должно быть определено |
| `{ f6 }` без text | `{ f6 { text "Возврат" } { pressed quit } }` | Каждая кнопка должна иметь текст |
| `{ f7 }` с горизонтальным меню | `{ f7 }` для вертикального ряда | f1-f6 — горизонтальный ряд, f7-f14 — вертикальный |
| `{ text "Текст" } { if условие text="Новый" }` | `{ text "Текст" { if условие text="Новый" } }` | Условие должно быть внутри блока text, а не после него |
