# HMI: Элементы — Item, Box, Rect, Line

> **⚠️ ВАЖНО ДЛЯ ИИ:**
> Элемент `{ item ... }` — основа любого HMI-экрана. Он может выводить как статический текст, так и значения переменных.
> **НЕ ИСПОЛЬЗУЙТЕ** `type text`, `field`, `label` или другие конструкции — в "Маяк-600" для всего используется `{ item }`.
> Все примеры в этом файле — точный синтаксис "Маяк-600".

## Item (элемент)

Команда индикации строки (вывод текста или значения переменной). Элемент `{ item ... }` является основным для вывода информации и организации ввода.

### Синтаксис

{ item <источник_данных> (атрибуты)
    <x> <y> <индикация> <нуль-индикация> '<формат>'
    { if <условие> ... }
    +<кнопка_увеличения> -<кнопка_уменьшения>
    !"<команда>"#"<подсказка>"
}

### Источник данных (имя элемента)

| Имя | Описание | Пример |
|-----|----------|--------|
| `#` | Пустое выражение (выводится только текст) | `#` |
| `d70` | D-ячейка (выводится значение) | `d70` |
| `m30` | M-переменная (выводится значение) | `m30` |
| `i34` | I-переменная (входы) | `i34` |
| `p11` | P-параметр | `p11` |
| `u12` | Параметр 0-299 | `u12` |
| `c124` | Параметр 300-399 | `c124` |
| `time` | Часы | `time` |
| `up` | Имя файла с УП | `up` |
| `comment` | Комментарий к УП | `comment` |
| `edit_name` | Имя редактируемого файла | `edit_name` |
| `txted_pos` | Позиция курсора в тексте | `txted_pos` |
| `txted_name` | Название редактируемого файла | `txted_name` |
| `diskfree` | Размер свободного дискового пространства | `diskfree` |
| `m32.0`, `m32.1` ... | Названия осей из справочника | `m32.0` |

**Доступ к битам и координатам:**
- `m32.0` — бит 0 M-переменной 32
- `i6.1` — бит 1 I-переменной 6
- `u0.0` — координата параметра u0
- `c15.1` — координата параметра c15

### Параметры элемента

| Параметр | Описание | Пример |
|----------|----------|--------|
| `height` | Высота строки (кегль) в пикселях | `height=20` |
| `width` | Ширина строки в пикселях (если не задана — авто) | `width=100` |
| `ratio` | Коэффициент сжатия шрифта (0.5, 0.6, 0.7, 1.0) | `ratio=0.5` |
| `fgr` | Цвет текста | `fgr=MAIN_TEXT` |
| `bgr` | Цвет фона | `bgr=MAIN_BGR` |
| `invfgr` | Цвет текста при инверсии | `invfgr=MAIN_BGR` |
| `invbgr` | Цвет фона при инверсии | `invbgr=CHANNEL0` |
| `align` | Выравнивание текста | `align=center` |
| `fonts` | Гарнитура шрифта | `fonts='IBMPlexMono'` |

**Константы выравнивания:**
- `TEXT_LEFT` / `LEFT` — по левому краю
- `TEXT_RIGHT` / `RIGHT` — по правому краю
- `TEXT_CENTERED` / `CENTER` — по центру
- `RIGHT+BOTTOM` — комбинация (правый нижний угол)
- `LEFT+BOTTOM` — комбинация (левый нижний угол)

**Специальные шрифты:**
- `'Mayak-Icons'` — иконочный шрифт для вывода символов ('c', 't', 'S', 't' и др.)

### Координаты

Координаты задаются в пикселях относительно левого верхнего угла View. Можно использовать:
- Числа: `0`, `10`, `456`
- Константы: `GAP_H`, `GAP_V`, `BOX_X`, `BOX_F_Y`
- Арифметические выражения: `BOX_X+BOX_GAP_H`, `BOX_F_Y+BOX_GAP_V+HEIGHT_MID+0.5`

### Индикация

- `yes` — элемент может быть выбран для ввода (при нажатии открывается наборная строка)
- `no` — элемент только для чтения

### Нуль-индикация

- `yes` — индицировать даже при нулевом значении переменной
- `no` — не индицировать при нулевом значении (элемент не отображается)

### Формат вывода (текст)

#### Простой текст (для источника #)
'Автомат'
'Фикс. перемещ.'
'Скорость: мм/мин'

#### Вывод из массива m32 (названия осей)
Синтаксис `'&N'`, где N — номер оси (0-5):
'&0'   ; выводит название оси 0 (например, "X")
'&1'   ; выводит название оси 1 (например, "Y")
'&2'   ; выводит название оси 2 (например, "Z")

#### Форматированный вывод чисел

| Формат | Описание | Пример | Результат |
|--------|----------|--------|----------|
| `[ ]` | Простое число | `[ ]` | 12345 |
| `[m]` | m знаков, дополнение нулями слева | `[5]` | 00123 |
| `[+m]` | m знаков с плюсом/минусом | `[+5]` | +0123 |
| `[m.n]` | m знаков всего, n после точки | `[9.3]` | 00012.345 |
| `[+m.n]` | с плюсом/минусом | `[+9.3]` | +0012.345 |
| `[m!n]` | отсечь n знаков справа | `[6!2]` | 00123 |
| `[m]%` | с символом процента | `[3]%` | 85% |

#### Вычисляемые выражения
`&(выражение)[]` — вычисление выражения:
'&(m28*1000/m29)[]'
'&(m17*1000)/d77[6.3]'

#### Вывод значения с индексом
'&m17[5]'   — значение M-переменной 17 в формате [5]
'&0[+9.3]'  — значение из массива с индексом 0

#### Системные форматы
| Формат | Описание |
|--------|----------|
| `<n>` | Текст из txtvals.dat по номеру n |
| `<!n>` | Текст из txtvals.dat (альтернативный формат) |
| `<!12>` | Текущее время |
| `<24>` | Статус отработки УП (m20) |
| `<32>` | Текст комментария |
| `<42>` | Сообщение D38/D39 (m28/m29) |

### Команды увеличения/уменьшения

Команды добавляют функциональность изменения значения переменной при нажатии клавиш →/←.

Синтаксис:
`+<команда_увеличения> -<команда_уменьшения>`

Можно указывать несколько пар команд:
`+ff10 -ff20 +ps1 -ps2`

**Стандартные команды:**
| Команда | Назначение |
|---------|------------|
| `+ff10 -ff20` | Увеличение/уменьшение процентов F |
| `+ps1 -ps2` | Увеличение/уменьшение процентов S |
| `+I10 -I20` | Увеличение/уменьшение координаты X |
| `+I11 -I21` | Увеличение/уменьшение координаты Y |
| `+I12 -I22` | Увеличение/уменьшение координаты Z |
| `+gs1 -gs2` | Переключение G90/G91 |
| `+gv1 -gv2` | Переключение G98/G99 |
| `+gp1 -gp2` | Переключение G17/G18/G19 |
| `+gi1 -gi2` | Переключение G0/G1/G2/G3 |
| `+modp03 -modm03` | Переключение модификаторов XL/XR/ZL/ZR |
| `+vcp01 -vcm01` | Переключение VC0/VC1 |
| `+vvp12 -vvm12` | Переключение VV1/VV2 |
| `+dpl -dmn` | Пользовательские действия |

**Специальные модификаторы:**
- `#StopIt` — остановка при достижении предела (для координат)
- `#StopNt` — остановка при достижении предела (для ручных режимов)

Пример:
{ item d65 (height=30) 516 66 no yes 'Позиция [1]' +dpl -dmn }
{ item i2 (height=FONTSIZE_SMALL) BOX_X+80+10 BOX_S_Y+BOX_GAP_V+HEIGHT_MID+0.5+4 no yes '[3]%' +ff10 -ff20 }

### Команды и подсказки

Синтаксис `!"<имя_действия>"#"<подсказка>"` позволяет вызвать действие, передав ему значение, введенное пользователем в наборную строку (`<ust>`).

Пример:
{ item # (height=20) 260 65 no yes '!"<ust>PF"#"<ust>"' }
Здесь при нажатии на элемент будет вызвано действие `PF` (определенное в `action.dat`), и ему в качестве параметра будет передано текущее содержимое наборной строки (`<ust>`).

### Условие (if)

Условия позволяют менять параметры элемента и текст в зависимости от значений переменных. Условия указываются в конце элемента.

Синтаксис:
{ if <условие> [атрибут=значение] [format='Текст'] }

**Операторы сравнения:**
| Синтаксис | Описание |
|-----------|----------|
| `=` | Равно |
| `#` | Не равно |
| `>` | Больше |
| `<` | Меньше |
| `>=` | Больше или равно |
| `<=` | Меньше или равно |

**Логические операторы:**
| Синтаксис | Описание | Пример |
|-----------|----------|--------|
| `&&` | И | `(u195=1)&&(m11=95)` |
| `\|\|` | ИЛИ | `(p7=0)\|\|(p30=p2)` |

**Битовые операции:**
| Синтаксис | Описание |
|-----------|----------|
| `p10&1#0` | Проверка 1-го бита |
| `p10&2#0` | Проверка 2-го бита |
| `p10&4#0` | Проверка 3-го бита |

**Ссылки на ячейки БПрО:**
| Тип | Синтаксис | Пример |
|-----|-----------|--------|
| D-ячейки | `dномер` | `d70` |
| Параметры 0-299 | `uномер` | `u12` |
| Параметры 300-399 | `cномер` | `c124` |
| Параметры 400-499 | `иномер.координата` | `u0.0` |
| Параметры 500-599 | `cномер.координата` | `c15.1` |
| Массив M | `mномер` | `m30` |
| Массив M с битом | `mномер.координата` | `m1.0` |
| Массив I | `iномер` | `i2` |
| Массив I с битом | `iномер.координата` | `i6.1` |
| Массив P | `pномер` | `p0` |

### Примеры item

#### 1. Простой вывод текста
{ item # (height=FONTSIZE_MID) GAP_H GAP_V yes yes 'Автомат' }

#### 2. Вывод значения переменной с форматированием
{ item d70 (height=20 width=50) 100 50 no yes '[5]' }

#### 3. Вывод названия оси из m32
{ item m32.0 (height=20) 100 50 no yes '&0' }

#### 4. Вывод с условием изменения цвета
{ item m20 (fgr=B3 width=160) 111 GAP_V no yes '<24>'
    { if m20=0 fgr=MAIN_BGR } }

#### 5. Вывод с условием изменения цвета и текста
{ item m30 (invbgr=Error_C invfgr=MAIN_BGR width=106)
    111 GAP_V yes yes 'ЧПУ: Авария'
    { if m30=255 invbgr=GREEN format='ЧПУ: Готово' } }

#### 6. Вывод с несколькими условиями (переключение единиц измерения)
{ item # (height=FONTSIZE_SMALL fgr=B3 bgr=B6 align=TEXT_RIGHT)
    BOX_X+BOX_GAP_H+80 BOX_F_Y+BOX_GAP_V no yes 'мм/мин'
    { if m11=95 format=' мм/об' }
    { if m11=941 format=' мкм/с' }
    { if m11=942 format='мм/час' }
    { if m11=943 format=' с/об' } }

#### 7. Вывод вычисляемого значения со сложным условием
{ item # (fgr=B3 bgr=B6)
    BOX_X+BOX_GAP_H+28 BOX_F_Y+BOX_GAP_V+HEIGHT_MID+0.5 no yes '&m17[5]'
    { if ((u195=1)&&(m11=95)&&(m6#0)&&(m6#4)&&(m6#28)&&(u294=0))
        format='&(m17*1000)/d77[6.3]' }
    { if ((u195=1)&&(m11=95)&&(m6#0)&&(m6#4)&&(m6#28)&&(u294=1))
        format='&(m17*1000)/i34[6.3]' } }

#### 8. Вывод с командой изменения значения
{ item d65 (height=30) 516 66 no yes 'Позиция [1]' +dpl -dmn }

#### 9. Вывод с несколькими командами
{ item d99 (height=20) 260 65 no yes '+ff1 -ff2 +df1 -df2' }

#### 10. Вывод с передачей значения из наборной строки
{ item # (height=20) 260 65 no yes '!"<ust>PF"#"<ust>"' }

#### 11. Вывод координаты с кнопками и условием
{ item m32.0 (height=FONTSIZE_BIG fgr=B1 width=SYM_10_BIG align=RIGHT+BOTTOM fonts='Mayak-Icons')
    GAP_H+D_COORD_1 BLOCK_COORD_Y+HEIGHT_MID+2 no yes '[+10.3] ' +I10#StopIt -I20#StopIt
    { if IND_KRD0#1 format='' } }

## Box (залитый прямоугольник)

Команда индикации залитого прямоугольника.

### Синтаксис
{ box color=<цвет> x=<X> y=<Y> x2=<ширина> y2=<высота> }
{ box color=<цвет> ifcolors='<цвет1> if <условие1>; <цвет2> if <условие2>' x=<X> y=<Y> x2=<ширина> y2=<высота> }

### Параметры
| Параметр | Описание |
|----------|----------|
| color | Цвет заливки (константа или RGB) |
| ifcolors | Условное изменение цвета (несколько условий через `;`) |
| x | Координата X левого верхнего угла |
| y | Координата Y левого верхнего угла |
| x2 | Ширина в пикселях |
| y2 | Высота в пикселях |

### Условный цвет (ifcolors)
Позволяет менять цвет прямоугольника в зависимости от условия:
{ box color=BOX_ZL_AVT_BGR_LOAD
    ifcolors='BOX_ZL_AVT_FGR_LOAD if (d710>0)&&(IND_KRD0=1); BOX_ZL_AVT_FGR_NO_LOAD if IND_KRD0=0'
    x=506 y=112 x2=142 y2=84 }
Синтаксис: `цвет if условие; цвет if условие` — несколько условий, разделенных `;`.

### Пример
{ box color=RGB(255, 205, 0) x=506 y=112 x2=142 y2=84 }
{ box color=B6 x=BOX_X y=BOX_F_Y x2=BOX_W y2=BOX_F_H }

## Rect (рамка прямоугольника)

{ rect color=<цвет> ifcolors='<цвет1> if <условие1>' x=<X> y=<Y> x2=<ширина> y2=<высота> }

### Параметры
| Параметр | Описание |

| x | Координата X левого верхнего угла |
| y | Координата Y левого верхнего угла |

| y2 | Высота в пикселях |

### Пример
{ rect color=RGB(255, 205, 0) x=300 y=10 x2=50 y2=25 }

## Plugin (встраиваемый модуль)

Специальный блок для вставки встроенных плагинов (редактор УП, осциллограф, графика и т.д.).

### Синтаксис
{ имя_плагина
    plugin_x = координата_X
    plugin_y = координата_Y
    plugin_w = ширина
    plugin_h = высота
    параметр1 = значение1
    параметр2 = значение2
}

### Основные плагины
| Имя плагина | Назначение |
|-------------|------------|
| `indicator_UP` | Индикатор текста УП |
| `GRAPH_AUTO` | Графика в режиме автомат |
| `GRAPH` | Графика |
| `OSCILLOSCOPE` | Осциллограф |

### Пример (индикатор УП)
{ indicator_UP
    plugin_x = BOX_UP_X
    plugin_y = BOX_UP_Y+24
    plugin_w = BOX_UP_W*1.15
    plugin_h = BOX_UP_H-24-BOX_UP_HEIGHT
    max_row = 16-2*IND_KRD0-2*IND_KRD1-2*IND_KRD2-2*IND_KRD3-2*IND_KRD4-2*IND_KRD5
    max_col = 50
}

### Пример (графика)
{ GRAPH_AUTO
    plugin_x=240 plugin_y=120 plugin_w=340 plugin_h=225
    max_row=12 max_col=30
}

## Dlgdata (табличное меню)

Блок `{ dlgdata ... } dlgdata` используется для создания табличных меню выбора. Он отображает список элементов из конфигурационного файла, позволяя оператору выбирать нужный пункт (например, G-код, цикл, геометрический элемент).

### Синтаксис
{ dlgdata menu=<номер_меню> cfgfile="<имя_файла>" columns=<количество_колонок> }
или
{ dlgdata (атрибуты) ... } dlgdata

### Атрибуты
| Атрибут | Описание | Пример |
|---------|----------|--------|
| menu | Номер меню, связанного с данным dlgdata | `menu=10000` |
| cfgfile | Имя файла конфигурации, содержащего список элементов | `cfgfile="ciklt.dat"` |
| columns | Количество колонок в таблице | `columns=1` |

### Использование

`dlgdata` обычно размещается внутри блока `grid` и комбинируется с обработчиком нажатия клавиши Enter:

{ grid g_dlgdata_1 ()
    { hotkey KEY_ENTER { pressed dataselect inpush "<?fun>" vvod_all } }
    { dlgdata menu=10000 cfgfile="ciklt.dat" columns=1 }
    { view (xmax=800 ymax=600 windowed=1 grid_y=0 grid_h=DEF_GRID_Y grid_selectable=0 bgr = BG_MAIN )
        { item # (height=FONTSIZE_MID ratio=0.5 invfgr=BG_MAIN invbgr=TEXT_MAIN)
            0 GAP_V yes yes ' Диалог: ввод цикла ' }
    } view
} grid

### Принцип работы

1. Конфигурационный файл (например, `ciklt.dat`) содержит список элементов, доступных для выбора
2. `dlgdata` отображает эти элементы в виде таблицы
3. Пользователь выбирает элемент с помощью курсора и нажимает Enter
4. Команда `dataselect` обрабатывает выбор, извлекает выбранный элемент и передает его в действие `vvod_all` (или другое) для дальнейшей обработки

### Пример с несколькими dlgdata для разных типов элементов

В реальных проектах часто создают несколько гридов с разными `dlgdata` для разных категорий:

; Для циклов
{ grid g_dlgdata_1 ()
    { hotkey KEY_ENTER { pressed dataselect inpush "<?fun>" vvod_all } }
    { dlgdata menu=10000 cfgfile="ciklt.dat" columns=1 }
    { view ... }
} grid

; Для G-функций
{ grid g_dlgdata_2 ()
    { hotkey KEY_ENTER { pressed dataselect inpush "<?fun>" vvod_all } }
    { dlgdata menu=10000 cfgfile="ciklt.dat" columns=1 }
    { view ... }
} grid

{ menu 10000

## Примечания

1. **Имена подключаемых файлов** (в примерах @clock.dat@, @box_name_s.dat@) зависят от конкретной реализации HMI. При работе с проектом необходимо ориентироваться на фактические имена файлов, указанные в коде.

2. **Константы** (GAP_H, GAP_V, BOX_X, FONTSIZE_MID и др.) определяются в файле CONFIG/04_CONSTANTS.md и могут отличаться в разных проектах.

3. **Цветовые константы** (MAIN_BGR, MAIN_TEXT, Error_C и др.) определяются в файле CONFIG/03_COLORS.md.

4. **Действия** (+ff10, +ps1, +dpl и др.) определяются в файлах action.dat или в коде через блоки { action ... }.

5. **Плагины** (indicator_UP, GRAPH_AUTO) являются встроенными модулями системы и их параметры могут различаться в зависимости от версии ПО.
