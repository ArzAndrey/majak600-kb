# HMI: Форматы вывода чисел и текста

> **⚠️ ВАЖНО ДЛЯ ИИ:**
> Формат вывода задаётся в одинарных кавычках после координат элемента `{ item ... }`.
> **НЕ ИСПОЛЬЗУЙТЕ** синтаксис форматирования из других ЧПУ — в "Маяк-600" используется свой набор спецификаторов.
> Все примеры в этом файле — точный синтаксис "Маяк-600".


## Зависимости
- HMI/01_SYNTAX_BASE.md — базовый синтаксис
- HMI/03_ELEMENTS.md — элемент item

---

## Общее описание

В элементе item формат вывода задаётся внутри кавычек после параметров. Формат определяет, как будет отображаться значение переменной или вычисленного выражения. Форматы можно комбинировать с текстом и несколькими значениями в одной строке.

---

## Форматы чисел

### [ ] — простое число без форматирования
Выводит число как есть, без дополнения нулями.

| Пример | Значение | Вывод |
|--------|----------|-------|
| [ ] | 12345 | 12345 |
| [ ] | -12 | -12 |

### [m] — фиксированная ширина, дополнение нулями слева
m — общее количество знаков (включая число). Применяется только для положительных чисел.

| Пример | Значение | Вывод |
|--------|----------|-------|
| [5] | 12345 | 12345 |
| [5] | 12 | 00012 |
| [5] | 0 | 00000 |

### [+m] — фиксированная ширина с плюсом/минусом
m — общее количество знаков, включая знак.

| Пример | Значение | Вывод |
|--------|----------|-------|
| [+7] | 12345 | +012345 |
| [+7] | -12345 | -012345 |
| [+7] | 0 | +000000 |

### [m.n] — фиксированная ширина с десятичной точкой
m — общее количество знаков (включая точку и цифры после неё)
n — количество знаков после точки
Применяется только для положительных чисел.

| Пример | Значение | Вывод |
|--------|----------|-------|
| [9.3] | 12345 | 00012.345 |
| [9.1] | 12345 | 0001234.5 |
| [7.2] | 12 | 00012.00 |

### [+m.n] — фиксированная ширина с плюсом/минусом и десятичной точкой
m — общее количество знаков (включая знак, точку и цифры после неё)
n — количество знаков после точки

| Пример | Значение | Вывод |
|--------|----------|-------|
| [+9.3] | 12345 | +0012.345 |
| [+9.3] | -12345 | -0012.345 |
| [+9.3] | 0 | +0000.000 |

### [m!n] — отсечение n знаков справа
m — общее количество знаков
n — количество знаков, отсекаемых с конца
Применяется только для положительных чисел.

| Пример | Значение | Вывод |
|--------|----------|-------|
| [6!2] | 12345 | 00123 |
| [6!1] | 12345 | 01234 |

### [+m!n] — отсечение n знаков справа с плюсом/минусом
m — общее количество знаков (включая знак)
n — количество знаков, отсекаемых с конца

| Пример | Значение | Вывод |
|--------|----------|-------|
| [+6!2] | 12345 | +00123 |
| [+6!2] | -12345 | -00123 |

## Вывод значений из массивов

### Синтаксис
'&<индекс>[формат]' — для вывода значения из массива по индексу
'&m<номер>[формат]' — для вывода значения M-переменной
'&d<номер>[формат]' — для вывода значения D-ячейки

### Примеры
| Формат | Описание |
|--------|----------|
| '&0[+9.3]' | Значение из массива с индексом 0 в формате [+9.3] |
| '&1[+9.3]' | Значение из массива с индексом 1 |
| '&m17[5]' | Значение M-переменной 17 в формате [5] |
| '&d77[6.3]' | Значение D-ячейки 77 в формате [6.3] |

---

## Вычисляемые выражения &()

### Синтаксис
&(выражение)[формат]

Выражение может содержать:
- числа
- имена переменных (d70, m30, u12, m1.0, i34 и т.д.)
- арифметические операции (+, -, *, /)
- скобки ()

### Примеры
| Формат | Значение | Вывод |
|--------|----------|-------|
| &(-568)[] | -568 | -568 |
| &(m28*1000/m29+55)[10.3] | m28=172, m29=10 | 000017.255 |

### Сложные вычисляемые выражения (из реальных проектов)
| Формат | Описание |
|--------|----------|
| &((m1.0-m6.0)*i34)/159159[3] | Вычисляется разность m1.0 и m6.0, умножается на i34, делится на 159159, результат выводится как целое число с тремя знаками |
| &((m1.0-m6.0)/159159)*d77[3] | Деление разности на константу, затем умножение на переменную d77 |
| &m92/d77[6.3] | Деление переменной m92 на d77, результат с 6 знаками всего, 3 после запятой |
| &m92/i34[6.3] | Деление переменной m92 на элемент массива i34 |
| &(m17*1000)/d77[6.3] | Умножение m17 на 1000, деление на d77 |
| &(m17*1000)/i34[6.3] | Умножение m17 на 1000, деление на i34 |

### Правила вычисления
1. Выражение вычисляется слева направо с учетом приоритета операций: сначала умножение/деление, затем сложение/вычитание.
2. Скобки () изменяют порядок вычислений.
3. Результат вычисления преобразуется в строку согласно указанному формату.

### Особенности
- Всегда выводится число или результат вычислений
- Игнорируется имя элемента (если указано d70, будет выведено &(...))

## Тексты из файла txtvals.dat

### Синтаксис
<n> — где n — номер текстовой строки в файле txtvals.dat
<!n> — альтернативный формат (используется для часов, имени файла и др.)

### Назначение
Позволяет выводить предопределённые текстовые сообщения (ошибки, предупреждения, системные сообщения, время, имя файла).

### Примеры из реальных проектов
| Формат | Значение | Вывод |
|--------|----------|-------|
| <51> | 51 | Не задана зона рассогласования при позиционировании |
| <25> | 25 | Не задана зона рассогласос |
| <!12> | - | 12:34:56 |
| <!55> | - | program.nc |
| <!80> | - | имя_файла.txt |
| <24> | m20 | Статус отработки УП |
| <32> | - | Текст комментария |
| <42> | m28/m29 | Сообщение D38/D39 |

### Обрезание текста
Если текст длиннее отведённого места, он обрезается по количеству символов, указанному в формате или ширине элемента.

## Специальные имена элементов для вывода

Вместо переменной можно использовать специальные имена для вывода системной информации.

| Имя | Назначение | Пример использования |
|-----|------------|----------------------|
| time | Текущее время (формат ЧЧ:ММ:СС) | { item time ... '<!12>' } |
| up | Имя текущего файла УП | { item up ... '<!55>' } |
| comment | Комментарий к УП | { item comment ... '<32>' } |
| edit_name | Имя редактируемого файла | { item edit_name ... } |
| txted_pos | Позиция курсора в текстовом редакторе | { item txted_pos ... '<!26>' } |
| txted_name | Название редактируемого файла | { item txted_name ... } |
| diskfree | Размер свободного дискового пространства | { item diskfree ... '[]кб' } |

### Примеры
{ item time (height=FONTSIZE_MID) 672 16 no yes '<!12>' }

{ item up (height=FONTSIZE_MID) 100 50 no yes '<!55>'
    { if m0#0 format='' }
}

{ item txted_pos (height=FONTSIZE_MID) 450 16 no yes '<!26>' }

{ item diskfree (height=FONTSIZE_MID) 500 16 no yes '[]кб' }

## Комбинирование форматов

Форматы можно комбинировать в одной строке. Можно выводить несколько значений, перемежая их текстом.

### Синтаксис
'Текст [формат] текст <n> текст &(выражение)[формат]'

### Примеры
| Формат | Значение | Вывод |
|--------|----------|-------|
| '![3] <51>' | d38=172 | !172 Не задана зона рассогласования при позиционировании |
| 'Скорость: [5] мм/мин' | d40=240 | Скорость: 00240 мм/мин |
| 'Ошибка: <25>' | d38=25 | Ошибка: Не задана зона рассогласос |
| 'Время: <!12>' | - | Время: 12:34:56 |
| 'X [ +9.3 ]' | m1.0=123.456 | X +00123.456 |

---

## Условное изменение формата

Формат может меняться в зависимости от значения переменной с помощью условия `{ if ... }`. Условия указываются после формата и могут заменять format полностью.

### Синтаксис
{ item переменная (параметры) x y inv always 'Текст [формат]'
    { if условие format='Новый текст [новый_формат]' }
}

### Примеры из реальных проектов
{ item d77 (height=20) 260 65 no yes ' '
    { if (i56=1)&&(u294=0) format='&((m1.0-m6.0)/159159)*d77[3]' }
    { if (i56=1)&&(u294=1) format='&((m1.0-m6.0)*i34)/159159[3]' }
}

{ item m20 (fgr=B3 width=160) 111 GAP_V no yes '<24>'
    { if m20=0 fgr=MAIN_BGR }
}

{ item d317 (height=20) 260 65 no yes '+I10#StopNt -I20#StopNt'
    { if ((u195=1)&&(m11=95)&&(m6#0)&&(m6#4)&&(m6#28)&&(u294=0)) format='[6.3]' }
}

{ item # (height=FONTSIZE_SMALL fgr=B3 bgr=B6 align=TEXT_RIGHT)
    BOX_X+BOX_GAP_H+80 BOX_F_Y+BOX_GAP_V no yes 'мм/мин'
    { if m11=95 format=' мм/об' }
    { if m11=941 format=' мкм/с' }
    { if m11=942 format='мм/час' }
    { if m11=943 format=' с/об' } }

## Полная таблица форматов

| Формат | Описание | Знак | Десятичная точка | Обрезание справа |
|--------|----------|------|------------------|------------------|
| [ ] | Простое число | + | нет | нет |
| [m] | Фикс. ширина | нет | нет | нет |
| [+m] | Фикс. ширина со знаком | + | нет | нет |
| [m.n] | Фикс. ширина с точкой | нет | + | нет |
| [+m.n] | Фикс. ширина со знаком и точкой | + | + | нет |
| [m!n] | Отсечение справа | нет | нет | + |
| [+m!n] | Отсечение справа со знаком | + | нет | + |
| &(выр)[] | Вычисляемое выражение | + | опционально | опционально |
| <n> | Текст из txtvals.dat | нет | нет | нет |
| <!n> | Текст из txtvals.dat (альт.) | нет | нет | нет |

---

## Особенности и рекомендации

1. **Форматы [m], [m.n], [m!n] работают только с положительными числами**
   Если значение отрицательное, формат может работать некорректно. Для отрицательных чисел используйте форматы со знаком [+m], [+m.n], [+m!n].

2. **Форматы [+m], [+m.n], [+m!n] работают с любыми числами**
   Автоматически добавляют знак + или - перед числом.

3. **При комбинировании форматов порядок не важен**
   'Текст [5] <51> &(d70)[3]' — все форматы будут обработаны правильно.

4. **Файл txtvals.dat**
   Содержит предопределённые текстовые сообщения. Номера сообщений соответствуют кодам ошибок и системным сообщениям. Файл может быть локализован под разные языки.

5. **Вычисляемые выражения**
   Могут быть сложными, включать несколько переменных и операций. Используйте скобки для явного указания порядка вычислений.

6. **Условное изменение формата**
   Позволяет динамически менять отображение в зависимости от состояния станка. Несколько условий обрабатываются последовательно.

7. **Специальные имена**
   time, up, comment, edit_name, txted_pos, txted_name, diskfree — выводят системную информацию без указания переменной.

8. **Ширина элемента**
   Если ширина элемента (width) меньше, чем длина отформатированной строки, текст будет обрезан.

## Пример комплексного item с форматированием

{ item d78 (height=FONTSIZE_MID width=150 fgr=MAIN_TEXT)
    100 50 no yes
    'Величина: [9.3] мм  (&(d78/25.4)[6.2] дюйм)'
}

Вывод при d78=12345: Величина: 00012.345 мм  (000486.02 дюйм)
