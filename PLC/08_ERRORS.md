# ПЭС: Ошибки и сообщения

## Зависимости
- PLC/01_SYNTAX_BASE.md — базовый синтаксис
- PLC/03_OPERATORS.md — операторы msg, warn, error

---

## Общее описание

В ПЭС предусмотрены средства для вывода сообщений, предупреждений и ошибок на экран УЧПУ. Сообщения отображаются в специальной области экрана и могут содержать форматированные значения переменных.

---

## Ячейки ошибок и сообщений

| Ячейка | Назначение |
|--------|------------|
| M28 | Код сообщения (информационные сообщения) |
| M29 | Код ошибки (остановка УП и ПЭС) |
| D38 | Код сообщения (альтернативный доступ) |
| D39 | Код ошибки (альтернативный доступ) |

### Диапазоны кодов
| Диапазон | Назначение |
|----------|------------|
| 0-199 | Системные сообщения и ошибки БПрО |
| 200-599 | Ошибки электроавтоматики (ПЭС) |
| 600-999 | Системные ошибки |

---

## Операторы вывода сообщений

### msg — информационное сообщение
Синтаксис:
msg "Текст" [выражения] [if условие]

Описание:
Выводит информационное сообщение на экран. Не останавливает выполнение программы.

Примеры:
msg "Программа запущена"
msg "Скорость: [5] мм/мин" d40
msg "Готовность: [3]" m30 if m30 = 255

### warn — предупреждение
Синтаксис:
warn "Текст" [выражения] [if условие]

Описание:
Выводит предупреждение на экран. Не останавливает выполнение программы.

Примеры:
warn "Превышена скорость подачи"
warn "Температура: [3] град" d70 if d70 > 80

### error — ошибка
Синтаксис:
error "Текст" [выражения] [if условие]

Описание:
Выводит сообщение об ошибке на экран. Устанавливает M29 в указанный код. Останавливает выполнение УП и ПЭС (кроме ПП11 и ПП20).

Примеры:
error "Ошибка привода X"
error "Авария: [3]" d39 if d39 = 0

### syserror — системная ошибка
Синтаксис:
syserror "Системная ошибка", выражения [if условие]

Описание:
Добавляет ошибку в список системных ошибок без выдачи на экран. Используется для внутренней диагностики.

Пример:
syserror "Потеря связи с приводом X", d70

## Форматирование в сообщениях

В сообщениях можно использовать те же форматы, что и в HMI:

| Формат | Описание | Пример | Результат |
|--------|----------|--------|----------|
| [] | Число без форматирования | [] | 12345 |
| [m] | m знаков, дополнение нулями | [5] | 00123 |
| [+m] | m знаков с плюсом/минусом | [+7] | +012345 |
| [m.n] | m знаков всего, n после точки | [9.3] | 00012.345 |
| [+m.n] | с плюсом/минусом и точкой | [+9.3] | +0012.345 |
| [m!n] | отсечь n знаков справа | [6!2] | 00123 |
| [+m!n] | отсечь со знаком | [+6!2] | +00123 |

Примеры:
msg "Ток: [5] А" i2
msg "Позиция: [+9.3] мм" m1.0
msg "Ошибка: <51>" d38

## Текстовые сообщения из файла txtvals.dat

### Синтаксис
<n>

### Описание
Выводит текст из файла txtvals.dat, соответствующий номеру n.

### Файл errors.cfg
Содержит соответствие кодов и текстов сообщений.

Пример файла errors.cfg:
; Продолжение таблицы %syserrors, m28, m29
...
204 Нет напряжения задания на входе привода шпинделя
205 Шпиндель не вращается
206 Станок не включен
315 Вкл.РАК 20сек=&d101[ ] сек
212 Жду температуры N5 \&d400=1 $ п12 $$ &u12[4.1]С $

### Использование
msg "<204>"              ; выведет "Нет напряжения задания на входе привода шпинделя"
msg "Ошибка: <204>"      ; выведет "Ошибка: Нет напряжения задания на входе привода шпинделя"
msg "Статус: <315>" d101 ; выведет "Статус: Вкл.РАК 20сек=10 сек"

## Коды ошибок ПЭС (200-599)

Рекомендуемые диапазоны для пользовательских ошибок:

| Диапазон | Назначение |
|----------|------------|
| 200-249 | Ошибки приводов и двигателей |
| 250-299 | Ошибки датчиков и конечных выключателей |
| 300-349 | Ошибки гидравлики и пневматики |
| 350-399 | Ошибки СОЖ и смазки |
| 400-449 | Ошибки инструмента и магазина |
| 450-499 | Ошибки шпинделя |
| 500-549 | Ошибки безопасности |
| 550-599 | Общие ошибки ПЭС |

## Примеры использования

### Пример 1: Проверка аварийного выключателя
:216 RT(5)
    if A19.0 = 0
        error "Аварийный останов станка"
        d10 = 0
        d49 = 0
        A1.0 = 0
    fi

### Пример 2: Контроль температуры
:1
    if d70 > 80
        warn "Превышена температура: [3] град" d70
    fi
    if d70 > 100
        error "Критическая температура: [3] град" d70
    fi
    wait(1000)
    goto 1

### Пример 3: Проверка готовности привода
:103
    d49 = 1
    wait(100)
    if A20.0 = 0
        error "Привод шпинделя не готов"
        d49 = 0
        kill
    fi
    kill

### Пример 4: Контроль давления
:1
    if A21.0 = 0
        warn "Нет давления в гидросистеме"
        d10 = 0
    else
        msg "Давление в норме"
    fi
    wait(500)
    goto 1

### Пример 5: Информационное сообщение о смене режима
:24
    msg "Режим изменён: [0]" m0
    kill

### Пример 6: Сообщение с форматированием
:9
    msg "S задана: [5] об/мин" d1
    kill

### Пример 7: Сообщение из txtvals.dat
:1
    if m30 = 255
        msg "<206>"    ; "Станок включен"
    else
        msg "<206>"    ; всё равно выведет "Станок включен" — текст по номеру
    fi
    wait(1000)
    goto 1

### Пример 8: Условный вывод ошибки
:205
    while A17.0 = 0
        if d40 = 0
            error "Датчик зоны останова не сработал"
            kill
        fi
        wait
    wend
    kill

## Форматы текстовых сообщений в errors.cfg

| Конструкция | Описание |
|-------------|----------|
| &d101[ ] | Вставить значение D101 без форматирования |
| &u12[4.1] | Вставить параметр 12 с форматированием [4.1] |
| $ | Перенос строки |
| $$ | Перенос строки с отступом |
| \ | Продолжение строки |

Пример из errors.cfg:
212 Жду температуры N5 \&d400=1 $ п12 $$ &u12[4.1]С $

Это сообщение:
- Если d400=0: "Жду температуры N5 п12 20.0С"
- Если d400=1: "Жду температуры N5 п12 20.0С"

---

## Рекомендации по использованию

1. **msg** — использовать для информационных сообщений (старт программы, завершение операции)

2. **warn** — использовать для не критических ситуаций (превышение температуры, низкое давление)

3. **error** — использовать только для критических ошибок, требующих остановки (авария, потеря сигнала)

4. **syserror** — использовать для внутренней диагностики, не требующей внимания оператора

5. Коды ошибок 200-599 резервируются для ПЭС, избегайте конфликтов с системными ошибками

6. При выводе сообщений используйте форматирование для улучшения читаемости

7. Сообщения из txtvals.dat позволяют легко локализовать интерфейс
