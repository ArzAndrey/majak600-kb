# ПЭС: Операторы

## Зависимости
- PLC/01_SYNTAX_BASE.md — базовый синтаксис
- PLC/02_VARIABLES.md — переменные

## Общее описание

Операторы ПЭС выполняют действия:
- Присвоение значений переменным
- Вызов и запуск подпрограмм
- Ожидание и задержки
- Условное выполнение
- Циклы
- Переходы
- Индикацию сообщений

---

## Оператор присвоения (=)

### Синтаксис
переменная = выражение

### Описание
Вычисляет выражение и присваивает результат переменной.

### Примеры
d10 = 1
d40 = d40 + 10
my_var = (d78 / 1000) + 5
A1.0 = m30 / 255
D1, 10 = 0               ; присвоение группе переменных
D1.4, 10 = 1             ; присвоение биту группы переменных
D1 & 3, 10 = 0           ; присвоение по маске группе переменных

## Оператор call (вызов подпрограммы с возвратом)

### Синтаксис
call номер_пп
call номер_пп : метка
call номер_пп [if условие]

### Описание
Вызывает подпрограмму. После завершения вызванной подпрограммы (kill) выполнение продолжается со следующего оператора.

### Примеры
call 100                 ; вызов подпрограммы 100
call 100 : start         ; вызов с метки start
call 100 if d10 = 1      ; вызов по условию

---

## Оператор run (запуск подпрограммы в фоне)

### Синтаксис
run номер_пп
run номер_пп : метка
run номер_пп [if условие]

### Описание
Запускает подпрограмму на фоновое исполнение. Не ждёт завершения. Может работать параллельно с основной программой.

### Примеры
run 88                  ; запуск фоновой подпрограммы
run 88 : search         ; запуск с метки search
run 88 if d10 = 1       ; запуск по условию

---

## Оператор kill (останов подпрограммы)

### Синтаксис
kill
kill номер_пп
kill [if условие]

### Описание
- kill — завершение текущей подпрограммы
- kill номер_пп — останов указанной фоновой подпрограммы

### Примеры
kill                    ; завершить текущую подпрограмму
kill 88                 ; остановить фоновую подпрограмму 88
kill if d10 = 0         ; завершить по условию

---

## Оператор wait (ожидание)

### Синтаксис
wait                     ; задержка на один такт
wait(время)              ; задержка на указанное время (мс)
wait метка               ; задержка на один такт с переходом на метку
wait(время) метка        ; задержка с переходом на метку
wait if условие          ; ожидание выполнения условия
wait(время) if условие   ; ожидание условия или таймаута
wait метка if условие else метка2

### Описание
Прерывает выполнение подпрограммы на указанное время или до выполнения условия.

### Примеры
wait                     ; задержка на один такт
wait(100)                ; задержка на 100 мс
wait H100                ; задержка на один такт, затем переход на H100
wait(100) H100           ; задержка 100 мс, затем переход на H100
wait if A17.0 = 1        ; ожидание пока A17.0 = 1
wait(1000) if m30 = 255  ; ожидание 1 сек или пока m30=255
wait H100 if d10 = 0 else H200

---

## Оператор xwait (ожидание с сбросом таймера)

### Синтаксис
xwait(время)
xwait метка
xwait(время) метка
xwait if условие
xwait(время) if условие

### Описание
Аналогичен wait, но при повторном использовании происходит сброс вложенного таймера.

### Примеры
xwait(100)               ; задержка с возможностью сброса
xwait H100 if d10 = 1    ; ожидание с переходом на метку

---

## Оператор stop (завершение подпрограммы)

### Синтаксис
stop
stop [if условие]

### Описание
Завершает выполнение текущей подпрограммы. Вызвавшая подпрограмма (call) продолжает выполнение.

### Примеры
stop
stop if d10 = 0

---

## Оператор goto (переход)

### Синтаксис
goto метка
goto метка [if условие]

### Описание
Безусловный или условный переход на указанную метку.

### Примеры
goto start
goto exit if d10 = 0

---

## Оператор if (условное выполнение)

### Синтаксис
if условие операторы fi
if условие операторы1 else операторы2 fi
if условие1 операторы1 elif условие2 операторы2 ... else операторыN fi

### Описание
Выполняет операторы в зависимости от условия.

### Примеры
; простое условие
if d10 = 1
    d49 = 1
fi

; условие с else
if d10 = 1
    d49 = 1
else
    d49 = 0
fi

; множественное условие
if d38 = 0
    d39 = 0
elif d38 = 1
    d39 = 100
elif d38 = 2
    d39 = 200
else
    d39 = 999
fi

## Оператор while (цикл)

### Синтаксис
while условие
    операторы
    [wait]
wend

### Описание
Выполняет операторы до тех пор, пока условие истинно. При длительном ожидании необходимо использовать wait.

### Примеры
; простой цикл
while d40 > 0
    d40 = d40 - 1
    wait
wend

; цикл с ожиданием условия
while A17.0 = 0
    wait(100)
wend

### break и continue
while условие1
    операторы1
    [wait]
    break [if условие2]
    операторы2
    continue [if условие3]
    операторы3
wend

break — прерывает выполнение цикла
continue — переходит к следующей итерации

## Операторы индикации сообщений

### msg "Текст" [выражения] [if условие]
Выводит информационное сообщение на экран.

### warn "Текст" [выражения] [if условие]
Выводит предупреждение на экран.

### error "Текст" [выражения] [if условие]
Выводит сообщение об ошибке на экран. Устанавливает M(29) в указанный код.

### syserror "Системная ошибка", выражения [if условие]
Добавляет ошибку в список системных ошибок без выдачи на экран.

### Форматирование в сообщениях
| Формат | Описание | Пример |
|--------|----------|--------|
| [] | Число без форматирования | [ ] |
| [m] | m знаков, дополнение нулями | [5] |
| [+m] | m знаков с плюсом/минусом | [+7] |
| [m.n] | m знаков всего, n после точки | [9.3] |
| [+m.n] | с плюсом/минусом и точкой | [+9.3] |
| [m!n] | отсечь n знаков справа | [6!2] |
| [+m!n] | отсечь со знаком | [+6!2] |

### Примеры
msg "Программа запущена"
msg "Скорость: [5] мм/мин" d40
warn "Превышена скорость" if d40 > R504.0
error "Ошибка привода X" if d39 = 0

## Оператор Lipcmd (команда меню)

### Синтаксис
Lipcmd 'команда_меню'

### Описание
Выполняет команду меню пульта УЧПУ. Должен выполняться однократно.

### Примеры
Lipcmd 'XYZREC'        ; запись координат
Lipcmd 'XYZREC_STOP'   ; остановка записи

---

## Операторы сохранения данных

### rsave()
Сохранение параметров. Выполнять однократно после изменения таблицы инструментов из ПЭС.

### tsave()
Сохранение таблицы инструментов. Выполнять однократно.

### reset()
Сброс ЧПУ.

## Полные примеры

### Пример 1: Фоновая программа с ожиданием
:1
    ; инициализация
    d40 = 100
    d49 = 0
    
    ; основной цикл
    while 1
        ; проверка кнопки пуска
        if A17.0 = 1
            d10 = 1
            d49 = 1
        fi
        
        ; проверка кнопки стоп
        if A18.0 = 1
            d10 = 0
            d49 = 0
        fi
        
        wait(100)
    wend

### Пример 2: Обработка M-функции
:103 P1005
    ; включение шпинделя
    d49 = 1
    
    ; ожидание готовности привода
    while A20.0 = 0
        wait(50)
    wend
    
    ; ожидание выхода на скорость
    wait(500)
    
    ; проверка ошибок
    if A21.0 = 1
        error "Ошибка привода шпинделя"
        d49 = 0
        kill
    fi
    
    kill

### Пример 3: Цикл с ожиданием условия
:205
    ; выход в ноль по координате X
    d78 = 1000           ; величина перемещения
    d40 = 240            ; скорость
    d47.0 = 1            ; направление "+X"
    
    ; ожидание окончания движения
    while d40 != 0
        wait
    wend
    
    ; проверка сигнала датчика
    if A17.0 = 0
        error "Датчик зоны останова не сработал"
        kill
    fi
    
    kill

### Пример 4: Использование ifok в меню (для HMI, но через Lipcmd)
; в action.dat
{ action toggle_speed
    if d40 = 100
        d40 = 200
    else
        d40 = 100
    fi
}

; в ПЭС (вызов через Lipcmd)
Lipcmd 'toggle_speed'
