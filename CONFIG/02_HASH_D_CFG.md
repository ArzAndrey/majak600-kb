# CONFIG: Файл hash_d.cfg

## Зависимости
- DATA/01_D_CELLS.md — D-ячейки
- DATA/03_ARRAYS.md — массивы I, M, P
- DATA/02_PARAMETERS.md — параметры R

---

## Общее описание

Файл hash_d.cfg устанавливает соответствие между D-ячейками и элементами массивов БПрО. Это позволяет:
- Обращаться к внутренним данным БПрО через D-ячейки
- Использовать упрощённый доступ к сложным структурам данных
- Обеспечивать совместимость с ранее разработанными ПЭС

---

## Структура файла

; комментарий
DX d(X)                    ; D-ячейка как независимая переменная
DX u(Y)                    ; привязка к одномерному массиву
DX u(Y.K)                  ; привязка к двумерному массиву

где:
- X — номер D-ячейки (0-999)
- u — имя массива (i, m, p, r, a, w, t)
- Y — номер элемента массива
- K — номер координаты или индекса

---

## Типы привязки

| Тип | Синтаксис | Описание |
|-----|-----------|----------|
| D-ячейка как переменная | DX d(X) | D-ячейка независима |
| Массив I | DX i(Y) или i(Y.K) | Данные интерполяции |
| Массив M | DX m(Y) или m(Y.K) | Данные индикации и станка |
| Массив P | DX p(Y) | Данные из УП |
| Параметры | DX r(Y) или r(Y.K) | Параметры станка |
| Входные адреса | DX a(Y) | Входные сигналы |
| Выходные адреса | DX a(Y) | Выходные сигналы |
| Таблица инструментов | DX t(Y.K) | Данные инструментов |

## Использование привязки в ПЭС

После установления привязки в hash_d.cfg, к данным можно обращаться как через D-ячейки, так и через исходные массивы.

### Пример 1: Обращение через D-ячейку
d10 = 1          ; запуск УП
d40 = 500        ; скорость по X
d78 = 10000      ; перемещение по X

### Пример 2: Обращение через исходный массив
i3 = 1           ; запуск УП
i1.0 = 500       ; скорость по X
i3.0 = 10000     ; перемещение по X

### Пример 3: Чтение состояния
if d16 = 6       ; УП отрабатывается
    ; ...
fi

if m20 = 6       ; то же самое
    ; ...
fi

## Создание собственных привязок

Разработчик может добавлять собственные привязки D-ячеек к любым элементам массивов.

### Синтаксис
DXXX i(Y)        ; привязка к одномерному массиву I
DXXX i(Y.K)      ; привязка к двумерному массиву I
DXXX m(Y)        ; привязка к одномерному массиву M
DXXX m(Y.K)      ; привязка к двумерному массиву M
DXXX p(Y)        ; привязка к массиву P
DXXX r(Y)        ; привязка к параметру
DXXX r(Y.K)      ; привязка к параметру с координатой
DXXX a(Y)        ; привязка к входному/выходному адресу
DXXX t(Y.K)      ; привязка к таблице инструментов

### Примеры пользовательских привязок
D210 u(102)      ; D210 соответствует параметру 102 (осцилляция по Z)
D211 u(120)      ; D211 соответствует параметру 120 (время смазки)
D212 u(133)      ; D212 соответствует параметру 133 (контроль давления)
D320 c(25.0)     ; D320 соответствует параметру 525 по X
D321 c(25.1)     ; D321 соответствует параметру 525 по Y
D322 c(25.2)     ; D322 соответствует параметру 525 по Z

## Рекомендации

1. **Одна D-ячейка может быть привязана только к одному элементу**

2. **Один элемент может быть привязан к нескольким D-ячейкам**

3. **D-ячейки D0-D104** используются для обмена с БПрО и системной ПЭС

4. **D-ячейки D115-D299** предназначены для внутреннего использования ПЭС

5. **D-ячейки D300-D449** используются системной ПЭС и расширенными функциями

6. **D-ячейки D2000-D2599** обеспечивают доступ к параметрам R0-R599

7. **При создании новых привязок** следите за уникальностью номеров D-ячеек

8. **После изменения hash_d.cfg** необходимо перезагрузить УЧПУ
